Suggested architecture = the economic option (Chat + Codex + 3-way analysis converge)

Home Grid can signal and decorate; AVANTI reasoning and actions happen in chat only.
  • Option B: static Home Grid catalog + deterministic API overlay. Banners show facts/status/CTAs — no LLM.
  • Only a few tiles get cached decoration; rename avantiAction → chatIntent and route reasoning into chat.
  • Free = deterministic only — no manual AVANTI, no LLM decoration.
  • Plus/Mentor decoration is TTL-cached + event-driven behind hard token budgets + feature flags.
  • History caching: store chat artifacts by ID, send a bounded context packet (~3–4K tok), never re-serialize full JSON every turn.

Build-gate before scale (non-negotiable)

  • Bedrock token/cost telemetry — capture input/output/cache tokens + est. cost per call (LangSmith shows $0 today = instrumentation gap, NOT free).
  • Route/feature tags + tier grouping + per-tier daily dashboards & caps.
  • Model governance + alert — log model per call, alert on unexpected model (we caught a silent Haiku 3.5 → 4.5 drift). Server-owned, allow-listed.
  • Cross-surface cache + budgets (mobile + GAME OS share the gateway).
  • Catalog drift guard: generate client catalog types from server schema / versioned /v1/home/context.
  • Deterministic state-mutating CTAs (reminder/schedule/payment) still need approval + idempotency + role gates.
Accuracy note: Pricing ($1/$5), system-prompt size (3,428 tok), and current volume (~86 convos/mo, ~$1–7/mo real) are verified/measured.
